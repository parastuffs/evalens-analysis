Gérer les modèles permet de spécifier les canvas de publication et des messages, ainsi que de maintenir leur historique. Ce processus permet aussi de consulter et filtrer les modèles, ainsi que d'y faire une recherche, et ce sur l'ensemble des informations les décrivant. Ce processus est du ressort du superviseur.<br/>Le processus garantit la cohérence de chaque modèle et respecte la règle rule-0007.<br/><br/>Chaque modèle est décrit par les informations suivantes :<br/><ul>
<li>Intitulé : texte ou descriptif du modèle</li><li>Description</li><li>Identifiant : permet d'identifier univoquement un modèle</li><li>Représentation visuelle : représentation utilisée en tant que canvas pour le modèle. Elle est formée sur base d'un texte et d'une typographie fournis par l'utilisateur, d'un lien vers un document externe ou d'une fonction programmée.</li><li>Domaine applicatif : spécifie la(les) application(s) dans la(les)quelle(s) le questionnire peut être utilisé</li><li>Propriétés : informations variables spécifique à chaque modèle, dépendant du domaine applicatif et du type.</li><li>Modifiable : le modèle peut être personnalisé</li><li>Type : type de modèle (message, publication, etc.)</li><li>Relations entre les modèles</li><li>Statut : publié, protype</li><li>Période de validité : période durant laquelle le modèle est utilisable.</li><li>Ordre d'apparition</li></ul>
<br/>Pour un modèle donné, ce processus fournit aussi les informations suivantes :<br/><ul>
<li>Objets en relation (ex : liste de distribution utilisée pour le modèle)</li></ul>
<br/>Pour faciliter la mise à jour, les opérations suivantes sont fournies :<br/><ul>
<li>Cloner un modèle existant</li><li>Définir un modèle comme base utilisable pour créer de nouveaux modèles</li><li>Périmer un modèle à la date du jour</li><li>Visualiser le modèle</li></ul><p/>
